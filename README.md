# Jepson eFlora — Offline Plant ID App

An offline-capable mobile web app for identifying California vascular plants using the **Jepson Manual** dichotomous keys. Works as a Progressive Web App (PWA) — install it to your home screen for a native app feel with no internet required after first load.

**→ [Open the App](https://jacksanfilippo.github.io/JepsonEflora/jepson_app_v2.html)**

---

## Features

| Tab | What it does |
|-----|-------------|
| 🔑 **Key** | Step-by-step dichotomous key — tap A or B at each couplet to identify your plant |
| 📋 **Families** | Browse all families and genera; search by name |
| 📝 **Notes** | Jot down field notes and glossary terms as you key |
| 🌿 **Saved** | Track plants you've identified — mark each as ✅ success, 🔍 uncertain, or ❌ failed |
| 🚩 **Errors** | Flag data errors in the key; copy them to share |

### Key navigation
1. Start at the **Key** tab — the app opens at the master groups key
2. Read both couplet options (A and B) and tap the one that matches your plant
3. Keep tapping until you reach a **result** — family, genus, or species
4. Tap **🌿 Save** to record the identification

### Install as a home screen app
- **iPhone/iPad**: tap the share icon → *Add to Home Screen*
- **Android**: tap the browser menu → *Add to Home Screen* or *Install app*

Once installed, the app works **completely offline**.

## Data

- Covers **California vascular plants** from the Jepson Manual (second edition)
- **112 families** with dichotomous keys
- **726 genera** with species-level keys
- **3,138 species** listed under their genera
- Plant data (~2 MB) is downloaded once on first visit and cached locally

> **Note:** This is an independent project. The key data was extracted from the Jepson Manual PDF via OCR and cleaned up programmatically. Some keys may contain extraction errors — use 🚩 Flag error to report them.

## Running locally

The app needs to be served over HTTP (not opened as a local file) because it fetches `jepson_data.gz` at startup.

**Option 1 — Python (simplest):**
```bash
cd /path/to/JepsonEflora
python3 -m http.server 8000
# then open http://localhost:8000/jepson_app_v2.html
```

**Option 2 — Node.js:**
```bash
npx serve .
# then open the URL it prints
```

**Option 3 — VS Code:** install the *Live Server* extension, right-click `jepson_app_v2.html` → *Open with Live Server*.

## Rebuilding from source

The app is generated by two Python scripts in the `Downloads/` folder (not in this repo):

| Script | Purpose |
|--------|---------|
| `extract_jepson.py` | Extracts dichotomous keys from the Jepson Manual PDF using PyMuPDF |
| `fix_keys.py` | Patches known OCR errors, synonym renames, dead-end destinations |
| `build_app.py` | Assembles `jepson_app_v2.html` + `jepson_data.gz` from CSVs |

**Build order:**
```bash
python3 extract_jepson.py   # generates jepson_data/*.csv from PDF
python3 fix_keys.py         # patches keys.csv in-place
python3 build_app.py        # outputs jepson_app_v2.html + jepson_data.gz
```

**Requirements:** Python 3.9+, `pymupdf` (`pip install pymupdf`)

## Known limitations

- **OCR artefacts:** some couplet text is garbled or truncated from the original PDF two-column layout
- **Synonym gaps:** older genus names (pre-2012 Jepson eFlora) may appear instead of current accepted names
- **Incomplete keys:** a small number of keys end early due to extraction failures — use 🚩 to flag these
- **Asteraceae / Poaceae:** very large families with complex internal group structures; navigation may be incomplete
